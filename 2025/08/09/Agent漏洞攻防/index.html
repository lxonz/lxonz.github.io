Cursor漏洞挖掘过程记录

发表于 2025-08-09 更新于 2025-08-15 分类于 LLM漏洞挖掘
本文字数： 722 阅读时长 ≈ 3 分钟

一、挖掘思路

感谢腾讯朱雀Nicky师傅、字节郁离歌师傅、漫雾T41nk师傅提供学习思路

首先得盘清楚，目前IDE侧的AI相关的漏洞都出现在哪些地方，因此找几个CVE与项目进行学习，例如：

  • CVE-2025-54135 （间接注入导致创建mcp.json文件的命令注入）Cursor
  • CVE-2025-54136 （通过 MCP 信任绕过实现RCE）Cursor
  • CVE-2025-54132 （通过 Mermaid 实现任意数据泄露）Cursor
  • CVE-2025-53773 （间接注入导致修改配置文件命令执行）Vscode
  • 漫雾MCP测试项目 https://github.com/slowmist/MasterMCP
  • 玄武在BlackHat USA揭示劫持智能体实现RCE的新方法 https://mp.weixin.qq.com/s/LHD9MAx5Yi43ylLlcQV4CQ
  • 国外专门挖LLM漏洞的师傅wuzzi的博客 https://embracethered.com/blog/index.html

二、过程记录

在前人的基础上对可能产生问题的地方进行测试。

在Web中进行间接注入

将提示词嵌入进html看是否有效果

嵌入的提示词，在检索后会检查到注释的内容，并会依据注释的内容去执行，尝试注入一些让模型进行命令执行的工作

确实执行命令了，虽然我只是让他访问网页的内容，他依据注释的内容进行了执行

在图片中进行间接注入

将提示词base64后使用010插入在文件头

证明藏在图片里还是可以识别到的，但识别到指令后，cursor判断不执行，那我们修改一下，看看能否让他做一些事情

打开计算器还是触发了，毕竟没有危害，那试试能不能弹个shell

在mcp.json里写入恶意mcp被发现并拦截，那试试能否直接写mcp.json，先伪装成正常的

还是失败了，cursor在几轮版本更新之后，显然变的更聪明了

目前还未找到更好的绕过方法，需要重新思考一下思路，确认攻击面：

  1. cursor存在指令白名单，是否能在白名单里进行曲线救国（cursor）
  2. 通过注入or其他方法修改cursor配置文件，打开一些危险选项（cursor）
  3. 提示词不管藏在web、图片或者音频，本质上只是输入媒介，还是需要想办法绕过cursor的安全机制（cursor）
  4. 假设cursor安全机制已经建立的比较完善，那需要从热门下载量的MCP下手（供应链打击）

先更新到这，等学习到MCP影响到IDE的内容在更新
